# m-main-notes

## 项目简介

`m-main-notes` 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面内容多样，涵盖笔记、工具、演示、文档等类型，彼此之间没有强关联。本仓库不针对任何特定域名或网站，仅作为个人或团队存放静态页面资源的统一位置。

## 目录说明

```
m-main-notes/
├── index.html          # 仓库总览页面（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/             # 公共资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md
```

- `pages/`：每个 HTML 文件代表一个独立页面，文件名应简洁、有辨识度。
- `assets/`：存放多个页面可能共用的资源文件，避免重复。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，每个页面可单独访问，不依赖后端服务。页面之间可能存在引用关系，但原则上每个页面应自包含（包括样式和脚本），或引用 `assets/` 下的公共资源。

- 页面命名建议：使用英文小写字母和连字符，例如 `project-overview.html`。
- 页面内容建议：在 `<head>` 中添加 `<meta charset="UTF-8">` 和 `<title>` 标签，以便正确显示和识别。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件，并确保所有引用路径正确。
- 更新页面：直接修改对应 HTML 文件，注意保持与其他页面的兼容性。
- 删除页面：移除对应文件，并检查是否有其他页面引用该文件，如有则需同步更新。
- 公共资源：若需修改 `assets/` 下的文件，请确保所有引用该资源的页面均能正常工作。

## 其他

本项目不涉及任何商业推广或特定服务，仅作为静态页面归档用途。欢迎任何人通过 Issue 或 Pull Request 提出改进建议或报告问题。
